Pipeline overview Click to expand the step-by-step methodology
1

Upload & normalize

2D MRI slice (PNG/JPG) is resized to 256x256, ImageNet-normalized for the segmentation backbone, and 224x224 for the classifiers.

->|
2

Cascade segmentation

Grayscale input is auto-detected and routed to the T1c specialist; multi-modal input goes to v3. 4-way TTA averaging + largest-component filter.

->|
3

3-classifier ensemble

CNN + ResNet50 transfer + ViT-hybrid run in parallel via ONNX. Epistemic = std across models, aleatoric = entropy of mean.

->|
4

Feature extraction

Deterministic radiology features: geometry, intensity, GLCM texture, morphology, mass effect, internal architecture, grade-evidence score.

->|
5

Layered LLM report

Pattern A polish, Pattern B citation-checked differential, Pattern C/D vision observer. Every LLM claim is validated against measured features.

BraTS 2020 (Bakas et al., 2018) LGG-MRI Segmentation (Buda et al., 2019) SMP UNet + ResNet34 (Iakubovskii, 2019) Grad-CAM (Selvaraju et al., 2017) Score-CAM / Occlusion (Wang et al., 2020) Llama 3.3 70B Instruct (Meta, 2024) Gemma 3 27B IT (Google DeepMind, 2025)

Pattern Detector

Checks each region against what healthy brains look like in a large self-supervised vision model.

Reconstruction Detector

A generative model tries to redraw each pixel from noise; this shows where it struggles most — usually unusual structure.

Asymmetry Detector

Compares the left and right sides of the brain. Highlights where one side differs from its mirror — common with unilateral lesions.

Some detectors aren't enabled on this server, so their heatmaps don't appear here. The Asymmetry Detector always runs; the Pattern Detector and Reconstruction Detector require GPU-backed deployments.
